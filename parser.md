# Parser Implementation Details

This document explains the functioning of the two parser implementations provided in this project: `ll1_parser.cpp` (Table-Driven) and `recursive_parser.cpp` (Recursive Descent). Both parsers are designed to process tokens generated by the scanner and verify the syntax of the mini-language.

---

## 1. Table-Driven LL(1) Parser (`ll1_parser.cpp`)

The table-driven parser uses an explicit stack and a pre-computed parse table to drive the syntax analysis.

### Key Components

- **Grammar Representation**:
    - **`Terminal` & `NonTerminal` Enums**: Clearly define all possible tokens and grammar variables.
    - **`Symbol` Struct**: A wrapper that can hold either a terminal or a non-terminal, used as the building block for the stack and production rules.
    - **`Production` Struct**: Stores the rule number, description, and the Right-Hand Side (RHS) symbols.

- **The Parse Table (`buildParseTable`)**:
    - This is a 2D mapping: `Table[NonTerminal][Terminal] -> ProductionNumber`.
    - It encodes the grammar's logic. For any given non-terminal at the top of the stack and a lookahead token from the input, the table tells the parser which rule to apply.

- **The Parsing Engine (`LL1Parser::parse`)**:
    - **Explicit Stack**: Uses `std::stack<Symbol>` to keep track of expected symbols. It starts by pushing the EOF marker (`$`) and the start symbol (`<program>`).
    - **The Loop**:
        1. **Match**: If the stack top is a **terminal** and matches the current input token, both are consumed.
        2. **Expand**: If the stack top is a **non-terminal**, the parser looks up the corresponding production in the table. It pops the non-terminal and pushes the RHS symbols of the rule onto the stack in **reverse order** (ensuring the leftmost symbol is on top).
        3. **Error**: If the terminal doesn't match or the table entry is empty, a syntax error is reported.

### Advantages
- Decouples the parsing logic from the grammar rules.
- Easy to automate (the table can be generated by tools).

## 2. Recursive Descent Parser (`recursive_parser.cpp`)

The recursive descent parser is a top-down parser where each non-terminal in the grammar is implemented as a dedicated C++ function.

### Key Components

- **Implicit Stack**: Instead of an explicit `std::stack`, it uses the **system call stack** through recursive function calls.
- **Predictive Logic**:
    - The parser looks at the current token (lookahead) to decide which function to call or which path to take within a function.
    - **`expect(TokenType, ...)`**: A helper function that ensures the current token matches the expected type and consumes it. If it doesn't match, it throws a syntax error.

- **Grammar Functions**:
    - **`parseProgram()`**: The entry point, looking for `BEGIN ... END`.
    - **`parseStmtList()`**: Uses a `while` loop to process multiple statements, checking the `isStmtFirst()` set to decide when to stop (handling the ε-production).
    - **`parseExpr()` & `parseExprTail()`**: Handles operator precedence and associativity (e.g., addition and subtraction) through right-recursive calls.

### Key Highlights
- **Human Readable**: The code structure closely mirrors the grammar, making it easy to debug and manually extend.
- **Indented Trace**: The implementation includes a `depth_` variable to print an indented "Parse Trace" to stdout, visually showing the derivation tree during execution.

---

## 3. Comparison of Approaches

| Feature | Table-Driven (`ll1_parser.cpp`) | Recursive Descent (`recursive_parser.cpp`) |
| :--- | :--- | :--- |
| **Logic** | Iterative (while loop + explicit stack) | Recursive (function calls) |
| **Grammar Storage** | Map/2D Array (Parse Table) | Hardcoded in function logic |
| **Flexibility** | Easier to change grammar by updating the table | Requires changing function code |
| **Trace Style** | Shows rule applications and matches | Shows nested function entries (derivation tree) |

## How to Run

### Table-Driven Parser
```bash
g++ -std=c++17 -o ll1_parser ll1_parser.cpp
./ll1_parser tokens.out
```

### Recursive Descent Parser
```bash
g++ -std=c++17 -o recursive_parser recursive_parser.cpp
./recursive_parser tokens.out
```
